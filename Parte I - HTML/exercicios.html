Parte I: HTML


Introdução

1.1. Os três tipos de códigos básicos de uma página de hipertexto são HTML, CSS e Javascript.

  • HTML: é uma linguagem de marcação esponsável por definir quais elementos serão exibidos em uma determinada página.
  • CSS: é uma linguagem de folhas de estilo responsável por definir como os elementos de conteúdo serão mostrados
  • Javascript: é uma linguagem de programação e apesar de não ser obrigatória possui um papeis fundamentais nos sites. Os dois principais são:
    • a modifição de contúedo mediante a ocorrência de algum evento;
    • carregar conteúdo do servidor web em segundo plano, permitindo atualizar partes da página sem a necessidade de requisitar toda a página novamente.

1.2. Uma mesma página pode estar vinculado a vários arquivos CSS e a vários arquivos Javascript. Por exemplo, é comum ter arquivos CSS separados para estilizar diferentes elementos da página, como botões, layout, tipografia, etc. Da mesma forma, diferentes arquivos JavaScript podem ser vinculados para funcionalidades específicas, como exibir janelas de mensagem, animar elementos ou atualizar partes da página sem exigir uma recarga completa da página.

1.3. É possível diferentes páginas de hipertexto terem vínculo com os mesmos arquivos Javascript e CSS. Na verdade, isso é uma prática bem comum, pois ao vincular várias páginas aos mesmos arquivos JavaScript e CSS, garante-se consistência na apresentação visual e funcionalidade em todo o site. Essa prática também promove uma gestão eficiente de código e manutenção, já que quaisquer atualizações ou alterações nos arquivos JavaScript e CSS compartilhados serão automaticamente refletidas em todas as páginas vinculadas.

1.4. As principais diferenças entre uma página hipertexto estática e uma dinâmica são:

    Página Hipertexto Estática:
  • Composta por arquivos pré-existentes que são simplesmente baixados do servidor web.
  • O conteúdo é fixo e não muda frequentemente.
  • As mudanças no conteúdo são feitas diretamente nos arquivos HTML.
  • Baixa frequência de manutenção.
  • Vantagens: Fácil de manter, baixa carga no servidor, tempos de carregamento rápidos.
  • Desvantagens: Interatividade limitada, incapacidade de exibir dados em tempo real, não adequado para conteúdo que muda com frequência.
    Página Hipertexto Dinâmica:

  • Algumas ou todas as páginas são geradas por uma aplicação web.
  • O conteúdo é gerado dinamicamente quando a página é solicitada.
  • O conteúdo relacionado ao HTML é frequentemente gerado dinamicamente.
  • Permite atualizações frequentes e exibição de dados em tempo real.
  • Vantagens: Interatividade, exibição de dados em tempo real, conteúdo personalizado, adequado para conteúdo dinâmico.
  • Desvantagens: Maior carga no servidor, vulnerabilidades de segurança potenciais, mais complexo de manter.

1.5. Quando uma solicitação HTTP é feita a uma página de hipertexto estática, o seguinte caminho é seguido:

  1. A solicitação é enviada do navegador do cliente para o servidor DNS para resolver o nome de domínio para um endereço IP.
  2. Uma vez obtido o endereço IP, a solicitação é enviada para o servidor web correspondente.
  3. O servidor web processa a solicitação e recupera a página de hipertexto estática.
  4. O servidor então envia a página de hipertexto de volta para o navegador do cliente como resposta à solicitação original.
